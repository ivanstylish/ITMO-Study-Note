1. main 函数（主程序入口）
- 用处：程序的起点和总调度，负责初始化环境、读取输入、校验输入、调用核心逻辑、输出结果和停机。

- main (主体逻辑)：
初始化栈指针 sp = 0x1000 并对齐。
从 MMIO 地址 0x80 读取输入参数 n。
校验 n：如果 n ≤ 0，跳转到 main__invalid。
如果 n > 0，在栈上保存返回地址 ra，调用 sum_even 函数。
sum_even 返回后，恢复 ra，释放栈空间，并跳转到 main__write_output。
main__invalid (子标签：非法输入处理)：
当 n ≤ 0 时执行。
将返回值寄存器 a0 设为 -1，表示输入不合法。
main__write_output (子标签：统一输出口)：
无论是正常计算返回，还是非法输入返回，最终都会汇聚到此标签。
将 a0 中的结果（计算结果、-1 或 0xCCCCCCCC）写入 MMIO 输出端口 0x84。
执行 halt 永久停机。
2. sum_even 函数（核心计算逻辑）
- 用处：计算 1 到 n 的偶数和。采用数学公式 S = m * (m+1)（其中 m = n / 2）代替循环累加，并包含嵌套调用演示和完整的溢出检测机制。

sum_even (主体逻辑)：
建栈帧：分配 12 字节栈空间，保存 ra、s0、s1（被调用者保存寄存器）。
保存参数：将 n 从 a0 移动到 s0 防止被覆盖。
嵌套调用：保存当前 ra，调用 is_even(n) 演示嵌套调用，返回后恢复 ra（未使用 is_even 的返回值）。
公式计算：计算 m = n/2，计算 m * (m+1)，将低 32 位结果暂存于 t1。
溢出判断：提取乘积高 32 位（mulh），若高位非 0 或低位结果符号为负（正正得负），则跳转至 sum_even__overflow。
sum_even__ok (子标签：正常结果路径)：
当没有溢出时执行。
将 t1 中的正确结果移动到返回值寄存器 a0。
无条件跳转到 sum_even__return。
sum_even__overflow (子标签：溢出处理路径)：
当结果超出 32 位有符号整数范围时执行。
利用位运算技巧（0xFFFFFFFF XOR 0x33333333）构造出 0xCCCCCCCC 并存入 a0 作为溢出标记。
sum_even__return (子标签：函数统一返回出口)：
正常和溢出路径的最终汇聚点。
恢复 s1、s0、ra 的值。
释放 12 字节栈帧。
通过 jr ra 返回 main 函数。
3. is_even 函数（奇偶校验逻辑）
- 用处：判断传入的参数是否为偶数。这是一个为了满足“嵌套调用”要求而存在的辅助函数，虽然核心公式不需要它，但很好地演示了栈在多层函数调用中的保护作用。

is_even (主体逻辑)：
计算 n % 2（使用 rem 取余指令）。
如果余数等于 0，跳转到 is_even__yes。
如果不等于 0（即奇数），将返回值 a0 设为 0，并直接返回。
is_even__yes (子标签：偶数路径)：
当 n 是偶数时执行。
将返回值 a0 设为 1。
返回调用者（sum_even）。

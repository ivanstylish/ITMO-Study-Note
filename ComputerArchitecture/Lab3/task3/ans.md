RLE（Run-Length Encoding，游程编码）的本质是：用"数量+符号"两个字节来代替连续重复的多个字节，当重复序列够长时可以压缩数据。
整个程序分三个阶段，用下面这张图来描述：
```
IO 0x80 ──读──→ [input_buf]  ──压缩──→ [output_buf]  ──写──→ IO 0x84

Phase 1: 读      Phase 2: RLE 编码       Phase 3: 输出
```

1. 阶段 1（读输入）：从内存映射 IO 端口 0x80 一次读一个字节存入内存缓冲区，遇到 \n 或读满 64 字节停止。D4 记录读到了多少个有效字符。
2. 阶段 2（核心：RLE 编码）：用"双指针"思路扫描输入缓冲区。

D2 保存当前 run 的字符，D1 保存它连续出现的次数
每次看下一个字符：如果和 D2 相同且 D1 < 9，就让 D1++
否则 flush：把 ('0'+D1) 和 D2 这两个字节写入输出缓冲区，然后用下一个字符开新的 run
上限 9：因为我们用一个 ASCII 数字来表示计数，'1'~'9' 刚好是 9 个状态，超过就必须断开

3. 阶段 3（输出）：从输出缓冲区逐字节读出，写到 IO 端口 0x84，遇到 null 终止。
溢出处理：如果压缩后的长度 + 1（null 终止符）≥ 64，说明输出缓冲区装不下，写单个 0xFF 错误标记。值得注意的是：RLE 对于完全没有重复的字符串（如 "ABC"）实际上会膨胀——3A4B4C 比 AAABBBBCCCC 短，但 1A1B1C 比 ABC 长。这是 RLE 的固有局限性。

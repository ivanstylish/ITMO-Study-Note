### 计算机架构备考知识点全汇总 (2026版)
Сводка знаний по архитектуре компьютера (2026)
第1讲：课程导论与系统工程基础
Лекция 1: Введение и основы системной инженерии
系统架构定义 (Определение архитектуры): 环境中系统的基本概念或属性，体现在其元素、关系、设计和演化原则中
。
软件密集型系统 (Software-intensive systems): 软件开发或集成是主导考虑因素的复杂系统
。
V模型 (V-диаграмма): 描述了从需求定义到开发，再到测试验证的系统全生命周期
。
第2讲：信息系统与控制系统
Лекция 2: Информационные и управляющие системы
信息系统 (Информационные системы): 以性能和吞吐量为优先级，常见于集群和云计算
。
控制系统/嵌入式 (Управляющие системы): 与物理世界交互，强调实时性（Real-time）——即在确定的时间内给出响应，而非单纯的“快”
。
利害关系人 (Stakeholders): 对系统有权益或需求的个人或组织
。
第3讲：继电器与早期计算
Лекция 3: Ещё не компьютеры. Реле
计算工具 (Инструменты расчета): 算盘（数字式）与对数计算尺（模拟式）的区别。对数尺利用物理刻度实现乘除法
。
继电器 (Реле): 电子计算机的前身，通过电磁线圈控制开关。具有“0”和“1”两种状态
。
RS触发器 (RS-триггер): 利用继电器实现的最简单的存储单元，能够保持之前的状态
。
第4讲：数字逻辑与PLC
Лекция 4: ПЛК. Цифровая схемотехника
布尔逻辑 (Булев базис): 基础逻辑门（与、或、非）以及全能门（与非 NAND、或非 NOR）
。
组合电路 (Комбинационные схемы): 输出仅取决于输入，如加法器（Adder）和多路复用器（MUX）
。
同步电路 (Синхронная схемотехника): 整个系统由统一的时钟信号驱动，解决了信号竞争问题
。
第5讲：硬件生产工艺
Лекция 5: Особенности производства аппаратуры
印刷电路板 (PCB/Печатная плата): 组件的物理和电气连接载体，分为插装和表面贴装（SMD）
。
集成电路制造 (Кремниевое производство): 从硅砂到单晶硅棒，再到晶圆（Wafer）和光刻（Photolithography）
。
芯片分级 (Chip Binning): 即使是同一块晶圆生产的芯片，也会根据质量被锁频或禁用部分内核，作为不同型号销售
。
第6讲：软硬件协同模型
Лекция 6: Hardware/Software. Модель вычислений
计算模型 (MoC): 定义计算执行的规则。包括状态机（FSM）、图灵机（Turing Machine）和λ演算（Lambda calculus）
。
软硬件协同设计 (HW/SW CoDesign): 同时设计硬件和软件，通过模型验证来缩短周期并降低集成风险
。
第7讲：通用处理器与冯·诺依曼架构
Лекция 7: Универсальный процессор. Машина фон Неймана
冯·诺依曼原则 (Принципы фон Неймана): 使用二进制、程序存储在内存中、指令顺序执行
。
ISA 与微架构:
ISA (指令集架构): 程序员看到的界面（如x86, ARM）
。
微架构 (Микроархитектура): 具体的硬件实现方案（寄存器、ALU、总线连接）
。
第8讲：指令集架构详解 (CISC)
Лекция 8: Память инструкций, DataPath, CISC
哈佛架构 vs. 普林斯顿架构: 哈佛架构将指令和数据存储物理分离，允许同时访问
。
CISC (复杂指令集): 旨在通过一条指令完成多个操作（如内存读取+计算），微代码（Microcode）常用于实现复杂的控制逻辑
。
第9讲：RISC 与性能极限
Лекция 9: RISC, Производительность
RISC (精简指令集): 统一指令格式，仅通过 Load/Store 访问内存，便于实现流水线（Pipeline）
。
性能障碍:
Power Wall (功耗墙): 散热限制。
Memory Wall (存储墙): 内存访问速度远慢于CPU频率增长
。
阿姆达尔定律 (Закон Амдала): 描述了程序并行化后的理论加速比极限
。
第10讲：输入输出系统 (I/O)
Лекция 10: Ввод-вывод
PMIO (端口映射): 使用专门的指令（in/out）访问IO设备
。
MMIO (内存映射): 将IO设备寄存器映射到普通的内存地址空间，统一处理
。
轮询 (Polling): CPU主动询问设备状态，简单但浪费资源
。
第11讲：多任务处理与中断
Лекция 11: Многозадачность. Система прерываний
中断 (Interrupts): 外部事件触发CPU暂停当前任务并跳转到处理程序（ISR）
。
多任务机制:
协作式 (Cooperative): 任务自愿交出CPU
。
抢占式 (Preemptive): 操作系统强制切换任务
。
DMA (直接内存访问): 允许外设直接与内存交换数据，无需CPU干预
。
第12讲：内存分层与RAM
Лекция 12: Иерархия памяти. RAM
存储层级 (Иерархия): 寄存器 > 缓存 > 内存 > 磁盘。层级越高，速度越快，价格越贵
。
RAM类型:
SRAM (静态): 速度快，用于缓存，靠晶体管状态存储
。
DRAM (动态): 密度高，容量大，靠电容存储，需要定期刷新（Refresh）
。
第13讲：高速缓存 (Cache)
Лекция 13: Кеш
局部性原理 (Принцип локальности): 时间局部性和空间局部性
。
映射策略: 直接映射（Direct Mapping）、全关联（Fully Associative）和组相联（N-Way Set Associative）
。
缓存一致性: 使用 MOESI 协议保证多核环境下数据的一致性
。
第14讲：分层风格与低级并行
Лекция 14: Уровневая организация. Низкоуровневый параллелизм
超标量 (Superscalar): 在运行时动态检查指令流，自动并行执行
。
VLIW (超长指令字): 依靠编译阶段确定的并行性，简化硬件
。
OSI模型: 作为系统分层组织的典型案例
。
第15讲：非冯·诺依曼处理器与FPGA
Лекция 15: Не фон Неймановские процессоры. ПЛИС
弗林分类法 (Классификация Флинна): SISD, SIMD (如GPU), MISD, MIMD
。
FPGA (可编程逻辑阵列): 逻辑由数据定义，通过 LUT (查找表) 实现任意逻辑功能
。
HLS (高层次综合): 将C/C++等高阶语言自动转换为硬件描述代码（HDL）
。
第16讲：分布式系统与分类法
Лекция 16: Распределённые системы. Классификация Дункана
邓肯分类法 (Классификация Дункана): 细化了并行架构，包括向量机、脉动阵列（Systolic arrays）等
。
数据流架构 (Dataflow): 指令执行由操作数是否准备就绪驱动，而非指令计数器
。
分布式系统: 基于 Actor 模型（如 Erlang）解决多节点通信、故障隔离和可扩展性问题
。
特别专题：NITTA 架构 (CGRA)
Спец. лекция: NITTA (CGRA)
CGRA (大粒度可重构架构): 结合了空间和时间计算，专为控制算法和信号处理设计
。
混合特性: 结合了 TTA（传输触发架构）、NISC（无指令集计算）和 VLIW 的特点
。

### 涵盖了典型的 三层架构（Controller、Service、Repository/Entity）：  
#### 控制层相关  
`@RestController` - 注解，用于将类标记为控制器，将其中的方法标记为处理请求的方法，并将其返回值转换为 JSON 格式的响应。(@Controller + @ResponseBody)  
Аннотации используются для обозначения класса как контроллера, обозначения его методов как методов, обрабатывающих запросы, и преобразования их возвращаемых значений в ответы в формате JSON (@Controller + @ResponseBody).  

`@RequestMapping` - 注解，用于映射请求路径到控制器方法。 Аннотации используются для сопоставления путей запросов с методами контроллера.  

`@PostMapping / @GetMapping` - 注解，用于映射 HTTP POST/GET 请求到控制器方法。 Аннотации используются для сопоставления HTTP POST/GET запросов с методами контроллера.  

`@DeleteMapping` - 处理 HTTP DELETE 请求，通常用于从数据库或资源中删除数据。 Обработка HTTP-запросов DELETE, как правило, для удаления данных из базы данных или ресурса.  

`@RequestBody` - 将前端传来的 JSON 字符串解析为 Java 对象。 Преобразовать JSON-строку, отправленную с фронтенда, в Java-объект.  

`@Valid` - 触发数据校验 Запустить проверку данных

#### 逻辑与配置:  
`@Service` - 标识这是一个业务逻辑类，会被 Spring 容器托管。 Это указывает на то, что данный класс является классом бизнес-логики и будет управляться контейнером Spring.  

`@Configuration` - 声明这是一个配置类，用于定义 Bean。 Это класс конфигурации, используемый для определения компонентов (beans).

`@Bean` - 将方法返回的对象注册为 Spring 容器中的一个 Bean。 Зарегистрируйте объект, возвращаемый методом, как Bean в контейнере Spring.

`@EnableWebSecurity` - 启用 Spring Security 的 Web 安全支持。 Включите поддержку веб-безопасности в Spring Security.

#### 数据库存储
- 接收数据: PointController 接收到前端传来的 PointDto 对象（包含 x, y, r）。

- 业务处理: PointService 调用 isHit 计算结果，并创建一个 Result 实体对象。

- 持久化:

  - Service 类中调用了 resultRepository.save(result)。

  - Spring Data JPA 接收到这个实体，根据类上的 @Entity 和 @Column 注解，自动生成 SQL INSERT 语句。

- 数据库执行: 数据库执行 SQL，并将数据存入 result 表中。由于使用了 @GeneratedValue(strategy = GenerationType.IDENTITY)，数据库会自动为该行生成一个唯一的 id。

- Получение данных: PointController получает объект PointDto (содержащий x, y, r) от фронтенда.

- Обработка бизнес-процессов: PointService вызывает метод isHit для вычисления результата и создает объект сущности Result.

- Сохранение данных:

- Класс Service вызывает метод resultRepository.save(result).

- Spring Data JPA получает эту сущность и автоматически генерирует SQL-запрос INSERT на основе аннотаций @Entity и @Column в классе.

- Выполнение запроса в базе данных: База данных выполняет SQL-запрос и сохраняет данные в результирующей таблице. Благодаря использованию @GeneratedValue(strategy = GenerationType.IDENTITY), база данных автоматически генерирует уникальный идентификатор для этой строки.
